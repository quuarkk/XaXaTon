# XaXaTon — Комплексное решение для анализа ПДн в файловом хранилище

Данный репозиторий содержит два взаимодополняющих скрипта для работы с датасетом файлов и поиска персональных данных в соответствии с требованиями 152-ФЗ.

## 📦 Состав решения

| Скрипт | Назначение |
|--------|------------|
| `organize_by_real_format_recursive.py` | Реорганизация файлов по их реальному формату (определение по магическим байтам) |
| `scan.py` | Сканирование файлов на наличие персональных данных (ПДн) с формированием детального отчёта |

---

## 🛠️ Скрипт 1: Организатор датасета

### Назначение

Приводит хаотичную структуру файлов к единообразному виду, сортируя файлы по **реальному формату** (определяемому по магическим байтам, а не по расширению). Это позволяет обнаружить подмены расширений и корректно классифицировать файлы для дальнейшего сканирования.

### Поддерживаемые форматы

| Реальный формат | Целевая папка | Расширения |
|----------------|---------------|-------------|
| PDF | `pdf/` | .pdf |
| ZIP/DOCX/XLSX | `archives/` | .zip, .docx, .xlsx, .pptx |
| OLE2 (DOC/XLS/PPT) | `office/` | .doc, .xls, .ppt |
| RTF | `office/` | .rtf |
| Изображения | `images/` | .png, .jpg, .jpeg, .gif, .bmp, .tif, .tiff |
| Видео | `video/` | .mp4, .avi, .ogv |
| HTML | `html/` | .html, .htm |
| XML/JSON/CSV | `data/` | .xml, .json, .csv, .parquet |
| Текст | `text/` | .txt, .md, .rst, .log |
| SQLite | `sqlite/` | .sqlite, .db |

### Особенности

- **Обнаружение подмен** — файл с расширением `.jpg`, но реально являющийся HTML, будет перемещён в `html/` с предупреждением
- **Извлечение архивов** — опционально распаковывает ZIP-архивы
- **Защита от коллизий** — автоматическое переименование при совпадении имён
- **Dry-run режим** — просмотр планируемых действий без изменений
- **Сохранение отчёта** — JSON-отчёт со всеми перемещениями

### Запуск

```bash
# Показать, что будет сделано (без реальных изменений)
python organize_by_real_format_recursive.py /path/to/dataset --dry-run

# Реально переместить файлы (внутри исходной директории)
python organize_by_real_format_recursive.py /path/to/dataset --mode move

# Скопировать в новую структуру (сохраняя оригиналы)
python organize_by_real_format_recursive.py /path/to/dataset --mode copy --output /new/location

# Извлечь ZIP-архивы в процессе
python organize_by_real_format_recursive.py /path/to/dataset --mode move --extract-archives

# Сохранить отчёт в JSON
python organize_by_real_format_recursive.py /path/to/dataset --mode move --report report.json

# Верифицировать уже организованную структуру
python organize_by_real_format_recursive.py /path/to/organized --verify
```

### Параметры

| Параметр | Описание |
|----------|----------|
| `--mode` | `move` (переместить), `copy` (скопировать), `symlink` (симлинки) |
| `--output` | Целевая директория (для copy/symlink) |
| `--dry-run` | Только показать, что будет сделано |
| `--extract-archives` | Извлекать ZIP-архивы |
| `--no-unknown` | Не перемещать файлы с неопределённым форматом |
| `--verify` | Верифицировать уже организованную структуру |
| `--report` | Сохранить отчёт в JSON |

## 🔍 Скрипт 2: Сканер персональных данных

### Назначение

Сканирует организованную структуру (папки `data/`, `text/`, `images/`) и находит файлы, содержащие персональные данные. Скрипт автоматически определяет уровень защищённости (УЗ) по 152-ФЗ и формирует детальный отчёт.

### Что обнаруживается (15+ категорий)

| Категория | Примеры | Способ обнаружения |
|-----------|---------|---------------------|
| **ФИО** | Иванов Иван Иванович | Regex + фильтр топонимов |
| **Email** | user@example.com | Regex |
| **Телефон** | +7(123)456-78-90, 89123456789 | Regex |
| **Дата рождения** | 12.05.1985 | Regex + контекст |
| **Адрес** | г. Москва, ул. Ленина, д. 1 | Индекс + контекст |
| **СНИЛС** | 123-456-789 01 | Regex + проверка контрольной суммы |
| **ИНН** | 123456789012 (10 или 12 цифр) | Regex + проверка контрольной суммы |
| **Паспорт РФ** | 1234 567890 | Regex + контекст |
| **Банковская карта** | 4111 1111 1111 1111 | Алгоритм Луна + контекст |
| **Банковский счёт** | 20 цифр с "р/с" | Regex |
| **БИК** | 9 цифр с "бик" | Regex |
| **Биометрия** | отпечатки, радужка, лицо | Ключевые слова |
| **Здоровье** | диагноз, анамнез, инвалидность | Ключевые слова |
| **Политика** | партия, политические взгляды | Ключевые слова |
| **Национальность/раса** | религия, вероисповедание | Ключевые слова |

### Уровни защищённости (УЗ)

На основе обнаруженных категорий скрипт присваивает уровень защищённости в соответствии с 152-ФЗ:

| Уровень | Критерии | Рекомендуемые меры |
|---------|----------|---------------------|
| **УЗ-1** | Биометрия или спец. категории (≥2 категорий или ≥5 находок) | Меры 1-го уровня: шифрование, аттестация ИС, строгий аудит |
| **УЗ-2** | Платёжные данные или гос. идентификаторы в значимом объёме | Меры 2-го уровня: шифрование данных, ограниченный доступ |
| **УЗ-3** | Гос. идентификаторы или значимые обычные ПДн | Меры 3-го уровня: разграничение доступа, резервное копирование |
| **УЗ-4** | Обычные ПДн в небольшом объёме | Базовые меры 4-го уровня: ограничение доступа, уведомление об инцидентах |

### Запуск

```bash
# Базовый запуск с указанием корневой папки
python scan.py "C:\path\to\organized\share"

# С указанием пути к Tesseract (Windows)
python scan.py "C:\path\to\share" "C:\Program Files\Tesseract-OCR\tesseract.exe"

# Использование стандартной папки (C:\PyProject\XaXaTon\ПДнDataset\share)
python scan.py
```

### Выходные файлы

| Файл | Содержание |
|------|------------|
| `result.csv` | Итоговый файл для сдачи (size, time, name) |
| `result_new.csv` | Альтернативный результат (если основной занят) |
| `scan_debug.csv` | Детальный отладка по каждому файлу (УЗ, найденные совпадения) |
| `pdn_report_YYYYMMDD_HHMMSS.md` | Markdown-отчёт с анализом ПДн по 152-ФЗ |
| `scan_all.log` | Полный лог процесса сканирования |
| `converted_images/` | Папка с JPG-копиями TIF-файлов |

### Формат result.csv

```csv
size,time,name
3068287,sep 26 18:31,CA01_01.tif
24983,sep 26 12:08,anketa_dms.txt
156234,sep 25 09:15,customers.csv
```

- `size` — размер файла в байтах
- `time` — дата модификации (формат: `месяц день часы:минуты`, строчные буквы, день без ведущего нуля)
- `name` — имя файла (basename)

### Формат Markdown-отчёта

Отчёт содержит:

1. **Общую статистику** — количество файлов с ПДн и без
2. **Сводку по уровням защищённости** — распределение файлов по УЗ
3. **Суммарные категории ПДн** — общее количество находок по каждому типу
4. **Детальный список файлов с ПДн** — для каждого файла:
   - Формат и размер
   - Уровень защищённости
   - Найденные категории с маскированными примерами
   - Рекомендации по обработке

**Пример секции отчёта:**

```markdown
### `data/customers.csv`
- Формат: CSV · Размер: 156,234 байт
- **УЗ: УЗ-3** · Находок: 4266
- Категории:
  - ФИО: 4266 экз — `Иванов Иван Иванович`, `Петров Петр Петрович`, `Сидорова Анна Сергеевна` (+4263)
  - Email: 2999 экз — `user***@example.com`, `client***@mail.ru` (+2997)
- Рекомендация: Государственные идентификаторы или значимые обычные ПДн. Меры 3-го уровня защищённости. Разграничение доступа, резервное копирование.
```

## 🚀 Полный рабочий процесс

### Шаг 1: Реорганизация датасета

```bash
# Перейти в директорию с исходным датасетом
cd C:\PyProject\XaXaTon\ПДнDataset

# Посмотреть, что будет перемещено
python organize_by_real_format_recursive.py share --dry-run

# Выполнить реорганизацию (перемещение файлов)
python organize_by_real_format_recursive.py share --mode move
```


## 🔧 Требования

- Python 3.8+
- Tesseract OCR (с русским языком)
- Для `organize_by_real_format_recursive.py`: стандартная библиотека
- Для `scan.py`:

```bash
pip install pandas opencv-python pillow pytesseract numpy
```

### Установка Tesseract OCR

**Windows:** https://github.com/UB-Mannheim/tesseract/wiki  
⚠️ При установке обязательно выбрать пакет языка **Russian**

**Linux:**
```bash
sudo apt install tesseract-ocr tesseract-ocr-rus
```
